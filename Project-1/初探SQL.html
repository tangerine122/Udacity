初探SQL

实体关系图(ERD)

电子表格是存储数据的方法之一
有时候我们需要大量的电子表格来存储不用来源的数据
使用ERD我们可以把这个电子表格关系可视化
ERD——Entity Relationship Diagram
实体关系图

实体关系图是查看数据库中数据的常用方式。


每个电子表格都会以一个表的形式呈现
在最上方可以看到表的名称
下方则列出了每列的列名
以及表与表之间的配合工作的方式

SQL是一门用于与数据库交互的语言
他可以查询一个表，也可以查询若干个表

为什么要要使用SQL

为了日后使用，几乎所有的应用都需要先存储数据

数据库的功能：
数据库存储信息以便之后访问
而SQL语言则允许分析师以及其他人访问这些信息

为什么使用数据库：
1.保证数据的完整性
2.可以快速访问数据
3.可以很容易的共享数据

数据库如何存储数据

数据库中的数据存储在类似于excel电子表格的表中
表具有更严格规则的行和列

每列必须有独一无二的列名
每一列中的所有数据都必须有相同的类型
这使得数据库表的分析操作十分简单

注意点：

数据库中的数据存储在类似于excel电子表格的表中
同一列中的所有数据必须符合数据类型
类类型一致是快速使用数据库的主要原因之一

语句类型

  • CREATE TABLE 是一个在数据库中创建新表的语句。
  • DROP TABLE 是删除数据库中表的语句。
  • SELECT 读取并显示数据。我们将这称为查询。
    重点为select语句

语句可以理解为撰写正常的SQL代码
语句会告诉数据库相对数据库进行什么操作

不过大多数分析工作并不需要在数据库中添加、删除或修改数据
分析工作需要的是浏览和操作数据，只是偶尔才更新基础数据源

SQL 语句是可以读取和处理数据的代码。但这不是一个真正的句子。 SQL 不区分大小写

可以将语句视为一段编写正确的的SQL代码，一种处理\读取数据库中所存储数据的方式。

查询语句select和from

读取数据表

问题：

  • 你想从什么数据里调出信息？
  • 你想从哪个数据集里调出什么元素？

select用于提醒查询要返回哪些列
from用于提醒查询在哪个表中查询

select子句告诉查询要从哪个表里读取哪些列
列名由逗号隔开，最后一个列名无需加逗号
也可以通过星号* 来调用所有的列

select和from子句都是必填项

规定查询格式

大写

大写命令（SELECT、FROM），小写查询中的其他内容
使得查询更容易读取。

表和变量名中不需要空格

通常在列名中使用下划线，避免使用空格

在Postgres中，如果列或表名中有空格，就需要使用双引号括住这些列/表名（例如：FROM \”Table Name\”）。在其他环境中，可能会使用方括号（例如：FROM [Table Name]）。

在查询中使用空格

SQL查询忽略空格，因此可以根据需要在代码之间添加尽可能多的空格和空行。

SQL不区分大小写

分号

根据SQL环境，查询结尾可能需要一个执行的分号

我们认为在么个语句的末尾添加一个分号，这样做可以在一次运行多个命令

LIMIT

许多查询的目的只是要看某个表是什么样的，希望扫一眼数据的前几行，大概了解自己关心哪些字段，想对字段进行什么操作。

LIMIT子句：显示数据表中前n行记录
LIMIT命令始终是查询的最后一部分

如果要查询的数据表数据非常巨大，数十亿行，读取时可能要等很久才能看到所要的结果
这时LIMIT子句，就会只读取该表的前几十行数据以便方便分析，和快速的读取

SELECT *

FROM orders

LIMIT 10;

显示orders数据表前十行的所有列数据

ORDER BY

ORDER BY语句按任意行排序表
与excel中过滤器进行排序类似
ORDER BY语句始终在SELECT和FROM语句之后，但位于LIMIT语句之前

ORDER BY 语句中的列之后可以添加DESC ，然后按降序排序
默认ORDER BY 是按升序排序的，排序方式为a到z

数据表默认是按id排序的

我们可以一次对多列进行排序，此时可以按照从左至右列出的列进行排序

SELECT *

FROM orders

ORDER BY occurred_at DESC, total_amt_usd DESC

LIMIT 5;

SELECT *

FROM orders

ORDER BY occurred_at, total_amt_usd

LIMIT 10;

WHERE

使用WHERE语句来为表格创建子集

WHERE子句能让你用一些具体的标准来筛选一系列结果

就像在excel里使用的筛选功能

WHERE子句位于FROM 后，在ORDER BY 或LIMIT之前

子句必须按照正确的顺序来写，否则查询会提示错误

WHERE语句中使用的常用的符号：

  • > 大于
  • < 小于
  • >= 大于或等于
  • <= 小于或等于
  • = 等于
  • != 不等于
SELECT *

FROM orders

WHERE gloss_amt_usd >= 1000

LIMIT 5;

SELECT *

FROM orders

WHERE total_amt_usd < 500

LIMIT 10;

比较运算符也可以处理非数字数据
等于或不等于形成了闭集，能让你挑选匹配或不匹配任意值的行
需要用单引号把值括起来

SELECT name, website, primary_poc

FROM accounts

WHERE name = 'Exxon Mobil';

算术运算符

派生列(derived column)可以由简单的算术或任意数量的高级运算得出
常用的运算包括：

  • 乘法*
  • 加法+
  • 减法-
  • 除法/

在SQL中使用算术运算符时，采用PEMDAS运算顺序

他们并不直接从基础数据中选出，而需要经过某种程度的操作
要更改系统默认生成的难看的列名?column?，需要在派生列这一行加上AS
在生成派生列的同时为其命名
命名惯例：不要用大写字母或空格

SELECT id, account_id, standard_amt_usd/standard_qty AS unit_price

FROM orders

LIMIT 10;

SELECT id, account_id,

poster_amt_usd/(standard_amt_usd + gloss_amt_usd + poster_amt_usd) AS post_per

FROM orders;

逻辑运算符

逻辑运算符包括：LIKE、IN、NOT、AND & BETWEEN、OR

  • LIKE 用于进行类似于使用WHERE和=的运算，但是是用于可能不知道自己想准确查找哪些内容的情况
  • IN 用于执行类似于使用WHERE和=的运算，但用于多个条件的情况
  • NOT 与IN和LIKE一起使用，用于选择NOT LIKE或NOT IN某个条件的所有行
  • AND & BETWEEN 用于组合所有条件必须为真的运算
  • OR 用于组合至少一个组合条件必须为真的运算

LIKE

LIKE运算符对于处理文本非常有用
我们将在WHERE 子句中使用LIKE
LIKE运算符经常与%一起使用。
%代表一个或任意数量的字符

SELECT name

FROM accounts

WHERE name LIKE 'C%';

SELECT name

FROM accounts

WHERE name LIKE '%one%';

SELECT name

FROM accounts

WHERE name LIKE '%s';

IN

IN函数能让你根据若干可能值来筛选数据
就像用比较运算符，IN 需要用单引号，把非值数据括起来，值数据则可直接输入

在括号内想要放多少值就放多少值，但每个不同的值之间，都得输入一个逗号

IN 运算符对于数字和文本列都很有用。
这个运算符可使我们使用=，但对于特定列的多个项目
可以检查我们要提取数据的一个、两个或多个列值，但这些都在同一个查询中

SELECT name, primary_poc, sales_rep_id

FROM accounts

WHERE name IN ('Walmart', 'Target', 'Nordstrom');

SELECT *

FROM web_events

WHERE channel IN ('organic', 'adwords');

NOT

NOT 运算符是一个非常有用的运算符，用于与之前介绍的两个运算符 IN 和 LIKE 一起运算

通过指定 NOT LIKE 或 NOT IN，我们可以查找到所有不符合特定条件的行。

SELECT name, primary_poc, sales_rep_id

FROM accounts

WHERE name NOT IN ('Walmart', 'Target', 'Nordstrom');

SELECT *

FROM web_events

WHERE channel NOT IN ('organic', 'adwords');

SELECT name

FROM accounts

WHERE name NOT LIKE 'C%';

SELECT name

FROM accounts

WHERE name NOT LIKE '%one%';

AND和BETWEEN

AND运算符用于WHERE语句中，用于一次考虑多个逻辑子句

可以使用AND 运算符将LIKE、IN和NOT 逻辑链接到一起

BETWEEN比AND语句更清楚一些，特别是AND语句的不同部分使用相同的列时，就可以使语句比较清晰。

WHERE column >=6 AND column <= 10

编写成一下样子会更好

WHERE column BETWEEN 6 AND 10

SELECT *

FROM orders

WHERE standard_qty > 1000 AND poster_qty = 0 AND gloss_qty = 0;

SELECT name

FROM accounts

WHERE name NOT LIKE 'C%' AND name LIKE '%s';

SELECT *

FROM web_events

WHERE channel IN ('organic', 'adwords') AND occurred_at BETWEEN '2016-01-01' AND '2017-01-01'

ORDER BY occurred_at DESC;

OR

于AND运算符类似，OR运算符可以组合多个语句
OR 是SQL语句中的逻辑运算符，可用于选择满足两个条件之一的行
原理和AND 类似，但是后者选出的是同时满足两个条件的行，前者取其中之一的行

两个逻辑判断只要有一个为真，就会有结果返回出来

可以使用 OR 运算符将 LIKE、IN、NOT、AND 和 BETWEEN逻辑连接到一起。

将多个运算组合到一起时，可能经常需要使用括号来确保我们要执行的逻辑能得到正确执行

OR还能借助于括号与其他运算符一起使用

SELECT id

FROM orders

WHERE gloss_qty > 4000 OR poster_qty > 4000;

SELECT *

FROM orders

WHERE standard_qty = 0 AND (gloss_qty > 1000 OR poster_qty > 1000);

SELECT *

FROM accounts

WHERE (name LIKE 'C%' OR name LIKE 'W%') 

           AND ((primary_poc LIKE '%ana%' OR primary_poc LIKE '%Ana%') 

           AND primary_poc NOT LIKE '%eana%');

移动平均值

移动平均值用于将数据线性化，以便更容易观察长期趋势，也不会因日常波动而错乱

移动平均值是一个最老也是最流行的技术分析工具。
若依次得到一组测定值时，按顺序取一定数量的数据并算得其全部算术平均值，得到的数据就叫做移动平均值。

移动平均值，就是指定时间段，对时间序列数据进行移动计算平均值
